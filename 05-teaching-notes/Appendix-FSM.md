# 附录：有限状态机（FSM）

> 2023 年考了 10 分大题，2024 年删掉了。但 2025 是否回归不确定——建议花半小时掌握基本套路。
> 有限状态机就像"自动售货机的大脑"：根据当前状态和输入，决定下一步做什么、输出什么。

---

## A.1 什么是有限状态机

一个有限状态机由五个部分组成：

$$M = (S, I, O, f, g)$$

| 符号 | 含义 | 通俗说法 |
|:----:|------|----------|
| $S$ | 有限状态集 | "机器的所有可能心情" |
| $I$ | 有限输入字母表 | "机器能接受哪些指令" |
| $O$ | 有限输出字母表 | "机器能说什么话" |
| $f$ | 状态转换函数：$S \times I \to S$ | "收到这个输入后，心情变成啥" |
| $g$ | 输出函数：$S \times I \to O$ | "在这个心情下收到这个输入，说啥" |

**直观理解**：一台自动售货机。
- 状态 S = {"等待投币", "已投币5元", "已投币10元"}
- 输入 I = {投5元, 投10元, 按可乐钮, 按雪碧钮, 按退币钮}
- 输出 O = {"谢谢惠顾", "吐出可乐", "吐出雪碧", "找零5元", "退币5元"}
- $f$：投币后状态变，按饮料后回"等待投币"
- $g$：按饮料钮时的输出取决于当前投了多少钱

## A.2 FSM 的三种表示法

### A.2.1 状态表

最直观的表格形式：

| 当前状态 | 输入 | 下一状态 | 输出 |
|:--------:|:----:|:--------:|:----:|
| S0 | a | S1 | 0 |
| S0 | b | S0 | 1 |
| S1 | a | S1 | 1 |
| S1 | b | S0 | 0 |

### A.2.2 状态图

用有向图表示。顶点 = 状态，边上的标注 = "输入/输出"。

```
状态图示例：
       a/0
    ┌────────┐
    │   ┌────┼───┐
    ▼   │    ▼   │
   [S0] │   [S1] │
    │   │    │   │
    └───┼─b/1┘   │
        └───b/0──┘
```

图中：

- 箭头 S0 → S1 上标 a/0：在 S0 收到 a，转到 S1，输出 0
- 箭头 S0 → S0 上标 b/1：在 S0 收到 b，留在 S0，输出 1

### A.2.3 转换函数 + 输出函数

纯数学写法：
$$f(S_0, a) = S_1, \quad g(S_0, a) = 0$$
$$f(S_0, b) = S_0, \quad g(S_0, b) = 1$$
$$f(S_1, a) = S_1, \quad g(S_1, a) = 1$$
$$f(S_1, b) = S_0, \quad g(S_1, b) = 0$$

**三种表示法可以互相转换**，考试中常给一种让你画另一种。

## A.3 设计一个 FSM：奇偶校验器（真题）

**2023 Q24（10分）**：设计一个奇偶校验器。

> 输入是一串 0 和 1。输出要求：如果到目前为止读到的 1 的个数是奇数，输出 1；如果是偶数，输出 0。

### Step 1：定义状态

```
S0：已读取的 1 的个数为偶数（初始状态）
S1：已读取的 1 的个数为奇数
```

两个状态就够了！因为系统只需要"记住"当前 1 的个数是奇还是偶。

### Step 2：状态转换函数

```
f(S0, 0) = S0    读到一个0 → 1的个数不变 → 状态不变
f(S0, 1) = S1    读到一个1 → 偶数变奇数
f(S1, 0) = S1    读到一个0 → 1的个数不变 → 状态不变
f(S1, 1) = S0    读到一个1 → 奇数变偶数
```

### Step 3：输出函数

```
g(S0, 0) = 0     在S0（偶数个1）→ 输出0
g(S0, 1) = 1     在S0读到1 → 立即输出1（现在1个1，奇数）
g(S1, 0) = 1     在S1（奇数个1）→ 输出1
g(S1, 1) = 0     在S1读到1 → 立即输出0（现在偶数个1）
```

### Step 4：画出状态图

```
       1/1
    ┌─────────┐
    │   ┌─────┼──────┐
    ▼   │     ▼      │
   [S0] │    [S1]    │
    │   │     │      │
    └───┼─0/0─┘      │
        └────1/0─────┘

标注含义：
  S0 → S0: 0/0  (偶数状态读0，仍偶数，输出0)
  S0 → S1: 1/1  (偶数状态读1，变奇数，输出1)
  S1 → S1: 0/1  (奇数状态读0，仍奇数，输出1)
  S1 → S0: 1/0  (奇数状态读1，变偶数，输出0)
```

### Step 5：验证（用输入 01101 测试）

```
初始: S0

步骤  S0 →(0/0)→ S0  读到0，输出0，仍S0
      S0 →(1/1)→ S1  读到1，输出1，变S1（1个1=奇数）
      S1 →(1/0)→ S0  读到1，输出0，变S0（2个1=偶数）
      S0 →(0/0)→ S0  读到0，输出0，仍S0
      S0 →(1/1)→ S1  读到1，输出1，变S1（3个1=奇数）

输出序列：0 1 0 0 1 ✓
```

## A.4 三种表示法互转

考试中可能给状态表让你画状态图，或给状态图让你写转换函数。

**状态表 → 状态图**：每行画一条有向边，起点=当前状态，终点=下一状态，标注=输入/输出。

**状态图 → 状态表**：每条带标注的有向边对应表中一行。

**状态图 → 转换函数**：直接读出每条边：
- $f$(起点, 输入) = 终点
- $g$(起点, 输入) = 输出

## A.5 设计 FSM 的通用步骤

```
1. 分析问题：需要"记住"什么？ → 确定状态
2. 列出所有可能状态：能区分所有不同情况的最少状态
3. 对每个状态和输入组合，确定下一状态（f）
4. 对每个状态和输入组合，确定输出（g）
5. 画出状态图/填写状态表
6. 用几个典型输入序列测试验证
```

**关键技巧**：FSM 的"记忆"是有限的——只能记住"属于哪个状态"。
所以设计时要问自己：**最少需要几种状态来区分所有不同情况？**
