# 点割集 & 边割集

---

## 一、点割集（Vertex Cut）

> 删掉这个**顶点集合**，图的连通分量数增加，且这个集合是**极小的**（少删任何一个顶点都不会增加连通分量）。

### 三个关键

| 要点 | 含义 |
|---|---|
| 删一组顶点 | 同时删掉它们以及连着的所有边 |
| 连通分量增加 | 图从一块碎成多块 |
| 极小性 | 这个集合里每个顶点都是必要的，少删一个就不行 |

### 例子

```
  a —— b —— c
  |     |     |
  d —— e —— f
```

- **{b, e}** 是点割集吗？删掉 b 和 e → a-d 一块，c-f 一块 → ✓ 碎了。但少删一个呢？只删 b，图仍通过 e 连通 → 所以 {b} 不是。{b, e} 满足极小性 ✓
- **{a, d}** 呢？删了它们，图还连通 → ✗ 不是
- **{b}** 在这个图里不是点割集（因为还有 e 连着）

### 点割集 vs 割点

| | 割点 | 点割集 |
|---|---|---|
| 删几个 | 1 个顶点 | 可以一组顶点 |
| 关系 | 割点是大小为 1 的点割集 | 点割集可以含多个顶点 |

> 割点是"单人点割集"。

---

## 二、边割集（Edge Cut）

> 删掉这个**边集合**，图的连通分量数增加，且这个集合是**极小的**。

### 三个关键

| 要点 | 含义 |
|---|---|
| 删一组边 | 只删边，顶点全留下 |
| 连通分量增加 | 图断开 |
| 极小性 | 这个集合里每条边都是必要的 |

### 同一个例子

```
  a —— b —— c
  |     |     |
  d —— e —— f
```

- **{(a,b), (b,c), (b,e)}** 是边割集吗？删这三条，b 完全孤立 → 碎了。少删一条呢？b 还连着 → 满足极小性 ✓
- **{(a,b), (a,d)}** 呢？删了 a 完全孤立 → 碎了。少删一条 a 还连着 → 极小 ✓。**但这是关联割集**（见下文）
- **{(b,e)}** 单独一条 → 删了之后只有 b 孤立，是边割集 ✓（同时也是割边/桥）

### 边割集 vs 割边（桥）

| | 割边/桥 | 边割集 |
|---|---|---|
| 删几条 | 1 条边 | 可以一组边 |
| 关系 | 桥是大小为 1 的边割集 | 边割集可以含多条边 |

---

## 三、常见考题：关联割集

> 删掉**与某个顶点关联的所有边**，如果图断开，这就是边割集。

上例中 {(a,b), (a,d)} = 删掉 a 的所有边 → a 孤立 → 连通分量增加 → 是边割集。这种就叫"关联割集"，是最常见的边割集。

---

## 四、对比总表

| | 点割集 | 边割集 |
|---|---|---|
| 删什么 | 一组顶点 + 它们的边 | 一组边（顶点留着） |
| 结果 | 连通分量增加 | 连通分量增加 |
| 极小性 | 少删一个就不行 | 少删一条就不行 |
| 大小为 1 时 | = 割点 | = 割边/桥 |
| 常见考法 | 关联割集、极小性判断 | 关联割集、回路与割集的关系 |

---

## 五、极小性判断技巧

**验证一个集合是不是割集，两步：**

```
1. 删掉 → 图碎了？（连通分量增加）
2. 从集合里随便放回一个 → 图又连上了？（极小性）
```

两步都满足才是割集。

---

**一句话：点割集 = 删一组顶点让图碎的极小集合；边割集 = 删一组边让图碎的极小集合。"割点"和"桥"是大小为一的特例。**
